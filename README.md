# Infolada for Home Assistant

Интеграция **Infolada** подключает Home Assistant к личному кабинету интернет-провайдера [ИнфоЛада](https://infolada.ru/) и создаёт сущности с балансом, тарифом, статусом и другими данными договора.

## Возможности

После настройки интеграция создаёт одно устройство личного кабинета и следующие сущности:

| Сущность | Описание |
| --- | --- |
| Баланс | Текущий баланс лицевого счёта |
| Текущий тариф | Название активного тарифного плана |
| Номер договора | Номер лицевого счёта / договора |
| Контрагент | Имя, на которое оформлен договор |
| Статус интернета | Текущее состояние доступа |
| Рекомендуемое пополнение | Сумма, которую рекомендует пополнить кабинет |
| Бонусы | Бонусный баланс (если доступен) |
| Остаток трафика | Остаток включённого трафика в МБ (если доступен) |
| Последнее обновление | Время последнего успешного опроса |

## Установка через HACS

1. Установите [HACS](https://hacs.xyz/).
2. Добавьте этот репозиторий как custom repository:
   - **Repository**: `thebestbaduser/infolada-homeassistant`
   - **Category**: Integration
3. Установите интеграцию **Infolada** через HACS.
4. Перезагрузите Home Assistant.

## Настройка

1. Откройте **Настройки → Устройства и службы → Добавить интеграцию**.
2. Найдите **Infolada**.
3. Введите логин и пароль от [личного кабинета](https://start.infolada.ru/auth?tab=portal).
4. Задайте интервал обновления (по умолчанию 6 часов).

## Как это работает

Интеграция использует тот же REST API, что и веб-кабинет `https://infolada.ru/lk/`:

- авторизация через `/lk/auth`;
- получение данных через `/api/v2/internet-contract`, `/api/v2/internet-account` и `/api/v2/user/list`.

Официального публичного API у провайдера нет, поэтому интеграция может перестать работать после изменений на стороне ИнфоЛады.

## Безопасность

Логин и пароль хранятся в конфигурации Home Assistant. Рекомендуется использовать отдельный пароль для личного кабинета и не давать интеграции права, которые вам не нужны.
